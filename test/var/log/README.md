# Test runtime log files

The `parallel_runtime_rspec.*.log` files in this directory are here on purpose.

They're test runtime logs generated by [`parallel_tests`](https://github.com/grosser/parallel_tests), printed as part of the CI workflow summary, and (for runs on the `main` branch) also uploaded as workflow artifacts for convenient analysis via the GitHub API.

When tests are executed in a CI workflow, the runtime log for the respective stack is fed to `bundle exec parallel_rspec` to assist `parallel_test` with optimal distribution of `_spec.rb` files across the given number of test processes, which helps ensure overall test execution time is as minimal as possible.

As tests, over time, get added or removed from the `_spec.rb` files, or as `_spec.rb` files themselves are added or removed, these logs should, for each stack, periodically be updated using median times from recent test runs' logs and updated here in the repository.

## Re-generating based on logs from recent runs

The CI workflow uploads runtime logs for runs on the `main` branch as artifacts. These can be downloaded using the GitHub API, and then a simple set of shell operations can compute a new result with averages.

### Preparation

> [!IMPORTANT]
> You need `datamash`, `ditto`, and `gh` (logged in), installed in your machine.

Ensure your number formatting locale is correct, and make a temp dir for the downloaded runtime log artifacts:

```ShellSession
user@host:log$ which datamash ditto gh
/opt/homebrew/bin/datamash
/usr/bin/ditto
/opt/homebrew/bin/gh
user@host:log$ gh auth status --active
窶ｦ
user@host:log$ export LC_NUMERIC=C
user@host:log$ logstmp=$(mktemp -d)
```

### Downloading and extraction

```ShellSession
user@host:log$ gh api '/repos/heroku/heroku-buildpack-php/actions/artifacts?name=parallel-test-runtime-logs&per_page=10&page=1' |
               jq -r '.artifacts[] | .id, .archive_download_url' |
               xargs -L2 -- bash -c 'gh api "$1" | ditto -k -x - "'$logstmp'/${0}"'
```

The `$logstmp` directory will now have ten subfolders (named after the artifact ID), each with run a run log file for each stack:

```ShellSession
user@host:log$ find "$logstmp" -name '*.log' -exec basename '{}' ';' | sort | uniq
parallel_runtime_rspec.heroku-22.log
parallel_runtime_rspec.heroku-24.log
```

### Updating runtime logs with calculated medians

> [!IMPORTANT]
> Ensure your current working directory is `test/var/log/`, or adjust the `> "$log"` redirect accordingly, for this step.

```ShellSession
user@host:log$ pwd
窶ｦ/heroku-buildpack-php/test/var/log
user@host:log$ for log in $(find "$logstmp" -name '*.log' -exec basename '{}' ';' | sort | uniq)
               do
                 sort "$logstmp"/*/"$log" |
                 datamash -t ':' groupby 1 median 2 |
                 datamash -t ':' cut 1 round 2 > "$log"
               done
```

> [!NOTE]
> `datamash` does not support `-R 0` to round to integers globally, so we have to pipe through it twice to get clean integer times that ideally help keep diffs smaller.

Then `git add` and `git commit` the updated `parallel_runtime_rspec.*.log` files.
